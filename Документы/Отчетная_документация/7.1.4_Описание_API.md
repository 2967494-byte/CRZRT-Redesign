## 1. ОБЩИЕ СВЕДЕНИЯ

### 1.1. Назначение

Настоящий документ содержит спецификацию программных интерфейсов (API) серверной части веб-сайта АО «Центр развития закупок РТ», реализованных в каталоге `/new_design/api/` на языке PHP 8.x.

API обеспечивает взаимодействие клиентской части (публичные страницы и CMS) с серверной логикой, базой данных MySQL и внешней системой Bitrix24 CRM.

### 1.2. Архитектурный стиль

- **Протокол:** HTTP/HTTPS
- **Формат данных:** JSON (`Content-Type: application/json; charset=utf-8`)
- **Стиль:** REST-like (без строгого REST, эндпоинты — PHP-скрипты)
- **Аутентификация:** PHP Session (cookie `PHPSESSID`) для защищённых операций
- **Кодировка:** UTF-8

### 1.3. Общие коды ответов HTTP

| Код | Значение | Когда возвращается |
|-----|----------|-------------------|
| 200 | OK | Успешный запрос |
| 204 | No Content | OPTIONS preflight |
| 400 | Bad Request | Некорректные параметры, JSON |
| 403 | Forbidden | Нет сессии / не авторизован |
| 405 | Method Not Allowed | Неверный HTTP-метод |
| 413 | Payload Too Large | Файл превышает лимит |
| 422 | Unprocessable Entity | Бизнес-ошибка (Bitrix, валидация) |
| 500 | Internal Server Error | Ошибка сервера, БД, GD |
| 502 | Bad Gateway | Ошибка внешнего сервиса (Bitrix CDN, VK) |

### 1.4. Общий формат ошибки

```json
{
  "success": false,
  "error": "Текстовое описание ошибки на русском языке",
  "details": {}
}
```

Поле `details` опционально и может содержать техническую информацию для администратора.

### 1.5. CORS

Для публичных эндпоинтов (`bitrix-lead-enroll.php`) поддерживается preflight `OPTIONS`. Основной сценарий — same-origin запросы с домена `zakupki.tatar`.

## 2. АУТЕНТИФИКАЦИЯ

### 2.1. `auth.php`

**Базовый путь:** `/api/auth.php`  
**Сессия:** `session_start()` при каждом запросе

#### 2.1.1. Проверка сессии

```
GET /api/auth.php?action=check
```

**Авторизация:** не требуется

**Ответ 200 (авторизован):**

```json
{
  "authenticated": true,
  "user": {
    "id": 1,
    "name": "Главный Админ",
    "role": "superadmin"
  }
}
```

**Ответ 200 (не авторизован):**

```json
{
  "authenticated": false
}
```

#### 2.1.2. Вход в систему

```
POST /api/auth.php?action=login
Content-Type: application/json
```

**Тело запроса:**

```json
{
  "email": "admin@crzrt.ru",
  "password": "********"
}
```

**Ответ 200 (успех):**

```json
{
  "success": true,
  "user": {
    "id": 1,
    "name": "Главный Админ",
    "email": "admin@crzrt.ru",
    "role": "superadmin"
  }
}
```

**Ответ 200 (ошибка):**

```json
{
  "success": false,
  "error": "Неверный email или пароль"
}
```

**Побочные эффекты:** устанавливаются `$_SESSION['user_id']`, `$_SESSION['user_role']`, `$_SESSION['user_name']`, cookie `PHPSESSID`.

#### 2.1.3. Выход

```
GET /api/auth.php?action=logout
```

**Ответ:**

```json
{
  "success": true
}
```

## 3. НАСТРОЙКИ И КОНТЕНТ

### 3.1. `settings.php`

**Базовый путь:** `/api/settings.php`  
**Назначение:** чтение и запись JSON-конфигурации страниц (таблица `settings`)

#### 3.1.1. Получение одной настройки

```
GET /api/settings.php?key={STORAGE_KEY}
```

**Авторизация:** не требуется (публичное чтение)

**Заголовки ответа:**

```
Cache-Control: no-store, no-cache, must-revalidate, max-age=0
Pragma: no-cache
Content-Type: application/json; charset=utf-8
```

**Параметры:**

| Параметр | Тип | Обязательный | Описание |
|----------|-----|--------------|----------|
| `key` | string | да | Ключ настройки, напр. `crzrt_main_page_data` |

**Ответ 200:** тело ответа — **сырой JSON** (не обёрнутый в объект). Если ключ не найден — `{}`.

**Пример:**

```
GET /api/settings.php?key=crzrt_main_page_data
```

#### 3.1.2. Получение всех настроек

```
GET /api/settings.php
```

**Ответ 200:**

```json
{
  "crzrt_main_page_data": { ... },
  "crzrt_obuchenie_page_data": { ... }
}
```

#### 3.1.3. Сохранение настройки

```
POST /api/settings.php
Content-Type: application/json
Cookie: PHPSESSID=...
```

**Авторизация:** требуется `$_SESSION['user_id']`

**Тело запроса:**

```json
{
  "key": "crzrt_main_page_data",
  "value": {
    "heroSlides": [],
    "serviceCards": []
  }
}
```

**Ответ 200 (успех):**

```json
{
  "success": true,
  "key": "crzrt_main_page_data",
  "size": 45678
}
```

**Ответ 403:**

```json
{
  "success": false,
  "error": "Несанкционированный доступ :("
}
```

**SQL:** `INSERT ... ON DUPLICATE KEY UPDATE` в таблицу `settings`.

#### 3.1.4. Реестр ключей (Storage Keys)

| Ключ | Назначение |
|------|------------|
| `crzrt_main_page_data` | Главная страница |
| `crzrt_ecp_page_data` | Лендинг ЭТП |
| `crzrt_consulting_page_data` | Консалтинг |
| `crzrt_support_page_data` | Сопровождение |
| `crzrt_obuchenie_page_data` | Обучение |
| `crzrt_knowledge_page_data` | Документы |
| `crzrt_news_page_data` | Новости |
| `crzrt_about_data` | О компании |
| `crzrt_contacts` | Контакты |
| `crzrt_education_data` | Legacy |
| `crzrt_consulting_data` | Legacy |

## 4. ЗАГРУЗКА ФАЙЛОВ

### 4.1. `upload.php` — изображения

**Метод:** `POST`  
**Авторизация:** требуется сессия

**Content-Type:** `application/json`

**Тело запроса:**

```json
{
  "dataUrl": "data:image/png;base64,iVBORw0KGgo...",
  "slot": "hero_0",
  "maxWidth": 1920,
  "maxHeight": 1080,
  "preserveSize": false
}
```

| Поле | Тип | По умолчанию | Описание |
|------|-----|--------------|----------|
| `dataUrl` | string | — | Base64 Data URL изображения |
| `slot` | string | `image` | Идентификатор слота для имени файла |
| `maxWidth` | int | 1920 | Макс. ширина (px) |
| `maxHeight` | int | 1080 | Макс. высота (px) |
| `preserveSize` | bool | false | Не уменьшать hero-баннеры |

**Обработка:**

1. Декодирование base64
2. Масштабирование (GD)
3. Конвертация в **WebP**
4. Итеративное сжатие до целевого размера (300 КБ – 8 МБ в зависимости от slot)
5. Сохранение в `uploads/landing/`

**Ответ 200:**

```json
{
  "success": true,
  "url": "uploads/landing/hero_0_20260703120000_a1b2c3.webp",
  "width": 1520,
  "height": 420,
  "size": 245760
}
```

**Ошибки:**

| error | HTTP |
|-------|------|
| Некорректный data URL | 400 |
| GD/WebP недоступны | 500 |
| Несанкционированный доступ | 403 |

### 4.2. `upload-file.php` — документы

**Метод:** `POST`  
**Content-Type:** `multipart/form-data`  
**Авторизация:** требуется сессия

**Поля формы:**

| Поле | Тип | Описание |
|------|-----|----------|
| `file` | file | Загружаемый файл |
| `slot` | string | Префикс имени (опционально) |

**Допустимые расширения:** pdf, doc, docx, xls, xlsx, zip, rar, rtf, txt, odt, ods

**Максимальный размер:** 20 МБ (20971520 байт)

**Ответ 200:**

```json
{
  "success": true,
  "url": "uploads/files/document_20260703120000_a1b2c3.pdf",
  "name": "Исходное_имя.pdf",
  "size": 1048576
}
```

## 5. ИНТЕГРАЦИЯ BITRIX24

return [

**Переменная окружения:** `BITRIX_WEBHOOK_LEAD_ADD` (приоритет над файлом)

### 5.2. `bitrix-lead-enroll.php` — заявка на курс

**Метод:** `POST`  
**Авторизация:** не требуется (публичная форма)

**Тело запроса:**

```json
{
  "name": "Иван Иванов",
  "phone": "+7 (999) 123-45-67",
  "email": "ivan@example.com",
  "company": "ООО Пример",
  "audienceType": "individual",
  "source": "site",
  "sourceLabel": "Сайт zakupki.tatar",
  "courseTitle": "Курс по 44-ФЗ",
  "dateFrom": "2026-07-06",
  "dateTo": "2026-07-10",
  "durationDays": 5
}
```

| Поле | Обязательное | Описание |
|------|--------------|----------|
| `name` | ✅ | Имя заявителя |
| `phone` | ✅ | Телефон |
| `email` | — | E-mail |
| `company` | — | Компания (для ЮЛ) |
| `audienceType` | — | `individual` / `legal` |
| `courseTitle` | — | Название курса |
| `dateFrom` | — | ISO дата начала |
| `dateTo` | — | ISO дата окончания |
| `durationDays` | — | Длительность |

**Ответ 200:**

```json
{
  "success": true,
  "leadId": 51700,
  "message": "Заявка принята"
}
```

**Поля лида в Bitrix24:**

- `TITLE`: «Заявка на курс: {courseTitle}»
- `NAME`, `PHONE`, `EMAIL`
- `UF_CRM_1782988591057` — дата начала (ДД.ММ.ГГГГ)
- `UF_CRM_1782988606167` — дата окончания
- `UF_CRM_1782988621676` — название мероприятия
- `COMMENTS` — служебная информация

### 5.3. `bitrix-lead-course.php` — новый курс из админки

**Метод:** `POST`  
**Авторизация:** требуется сессия CMS

**Тело запроса:**

```json
{
  "title": "Очный курс по 44-ФЗ",
  "dateFrom": "2026-07-06",
  "dateTo": "2026-07-10",
  "durationDays": 5,
  "format": "och",
  "price": "23 700 ₽"
}
```

| Поле | Обязательное | Описание |
|------|--------------|----------|
| `title` | ✅ | Название курса |
| `dateFrom` | ✅ | ISO дата |
| `format` | — | `och` / `dist` |
| `price` | — | Стоимость |

**Ответ 200:**

```json
{
  "success": true,
  "leadId": 51698,
  "message": "Лид в Bitrix24 создан"
}
```

**Поля лида:**

- `TITLE`: «Мероприятие: {title}»
- `COMMENTS`: «Курс создан в админке сайта zakupki.tatar», формат, стоимость

### 5.4. `bitrix-form-meta.php` — метаданные CRM-формы

**Метод:** `GET`  
**Авторизация:** не требуется

```
GET /api/bitrix-form-meta.php?id=1048
```

**Назначение:** парсинг `loader_{id}.js` с CDN Bitrix24 для получения полей формы.

**Ответ 200 (успех):**

```json
{
  "success": true,
  "formId": 1048,
  "fields": {
    "name": "LEAD_NAME",
    "phone": "LEAD_PHONE",
    "email": "LEAD_EMAIL"
  },
  "hasCaptcha": true
}
```

### 5.5. `bitrix-enroll.php` (legacy)

Устаревший эндпоинт интеграции через `crm.site.form.fill`. Сохранён для обратной совместимости. **Рекомендуется** использовать `bitrix-lead-enroll.php`.

### 5.6. Библиотека `bitrix-lead-lib.php`

Не является HTTP-эндпоинтом. Содержит функции:

| Функция | Описание |
|---------|----------|
| `bitrix_load_config()` | Загрузка webhook URL |
| `bitrix_format_date_dmY($iso)` | Конвертация даты в ДД.ММ.ГГГГ |
| `bitrix_course_end_iso(...)` | Расчёт даты окончания курса |
| `bitrix_build_lead_fields($input)` | Формирование полей лида |
| `bitrix_lead_add($fields)` | POST crm.lead.add через cURL |

## 6. ВСПОМОГАТЕЛЬНЫЕ ЭНДПОИНТЫ

### 6.1. `video-thumb.php` — превью VK Video

```
GET /api/video-thumb.php?url={VK_VIDEO_URL}
GET /api/video-thumb.php?url={URL}&proxy=1
```

**Параметры:**

| Параметр | Описание |
|----------|----------|
| `url` | Ссылка на видео VK |
| `proxy` | `1` — отдать изображение напрямую |

**Ответ 200 (JSON):**

```json
{
  "success": true,
  "thumbnail": "https://...",
  "cached": true
}
```

### 6.2. `install.php` — инициализация БД

**Метод:** GET (выполняется в браузере один раз)

**Действия:**

- CREATE TABLE `users`, `settings`
- Создание/сброс пользователя `admin@crzrt.ru`

> **Внимание:** после установки ограничьте доступ к `install.php` на уровне веб-сервера.

### 6.3. `upgrade.php` — миграция схемы

**Метод:** GET

**Действие:** `ALTER TABLE settings MODIFY setting_value LONGTEXT`

## 7. ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ (JavaScript)

### 7.1. Загрузка контента страницы

```javascript
const STORAGE_KEY = 'crzrt_main_page_data';
const response = await fetch(`api/settings.php?key=${encodeURIComponent(STORAGE_KEY)}`);
const pageData = await response.json();
```

### 7.2. Сохранение из админки

```javascript
await fetch('api/settings.php', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  credentials: 'same-origin',
  body: JSON.stringify({ key: STORAGE_KEY, value: pageData })
});
```

### 7.3. Загрузка изображения

```javascript
const dataUrl = canvas.toDataURL('image/webp', 0.92);
const response = await fetch('api/upload.php', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  credentials: 'same-origin',
  body: JSON.stringify({ dataUrl, slot: 'hero_0', preserveSize: true })
});
const { url } = await response.json();
```

### 7.4. Заявка на курс

```javascript
await fetch('api/bitrix-lead-enroll.php', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({ name, phone, email, courseTitle, dateFrom })
});
```

## 8. БЕЗОПАСНОСТЬ API

| Мера | Реализация |
|------|------------|
| SQL Injection | PDO prepared statements |
| XSS | JSON-ответы, экранирование на клиенте |
| Auth | PHP Session для POST upload, settings, bitrix-lead-course |
| File upload | Whitelist расширений, random имена, лимиты размера |
| Secrets | `bitrix-config.local.php` вне git |

**Рекомендации (ТЗ):**

- CSRF-токены для POST-запросов
- Rate limiting на `auth.php` и `bitrix-lead-enroll.php`
- IP whitelist для `admin.html`
