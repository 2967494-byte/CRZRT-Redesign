## 2. НАЧАЛО РАБОТЫ

### 2.1. Вход в систему управления контентом

1. Откройте браузер (рекомендуется Google Chrome или Яндекс.Браузер)
2. Перейдите по адресу: **https://zakupki.tatar/new_design/admin.html**
3. В появившемся окне введите **E-mail** и **Пароль**, выданные администратором
4. Нажмите **«Войти»**
5. Дождитесь загрузки бокового меню слева

> Если появляется сообщение «Неверный e-mail или пароль» — обратитесь к системному администратору для сброса пароля. Не пытайтесь угадывать пароль многократно (система может временно заблокировать доступ).

### 2.2. Первый вход: что проверить

После успешного входа рекомендуется:

1. Убедиться, что отображается раздел **«Главная страница»**
2. Прокрутить область редактирования — данные должны подгрузиться
3. Открыть публичный сайт в новой вкладке: `index.html`
4. Сравнить содержимое с тем, что видно в CMS

### 2.3. Выход из системы

По окончании работы закройте вкладку браузера или выполните выход (если кнопка доступна). На общих компьютерах **обязательно** завершайте сессию.

## 3. ОБЩИЕ ПРИНЦИПЫ РАБОТЫ В CMS

### 3.1. Золотое правило: «Редактируй → Сохраняй → Проверяй»

Любое изменение в CMS проходит три обязательных этапа:

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 1. РЕДАКТ.  │ ──► │ 2. СОХРАНИТЬ│ ──► │ 3. ПРОВЕРИТЬ│
│  поля CMS   │     │  кнопку     │     │  на сайте   │
└─────────────┘     └─────────────┘     └─────────────┘
```

**Без нажатия «Сохранить изменения» ваши правки не попадут на сайт!**

### 3.2. Навигация по разделам

- Щёлкните нужный пункт в **левом боковом меню**
- Центральная область обновится формой редактирования
- Активный пункт меню подсвечивается

### 3.3. Типы полей ввода

| Тип поля | Внешний вид | Как заполнять |
|----------|-------------|---------------|
| Текстовое | Однострочное поле | Введите текст, Enter не сохраняет |
| Многострочное | Большое текстовое поле | Тексты, описания; перенос строки — Enter |
| WYSIWYG | Редактор с панелью кнопок | Форматированный текст, как в Word |
| Изображение | Превью + кнопка «Загрузить» | Выберите файл, обрежьте, подтвердите |
| Чекбокс | ☐ / ☑ | Включить/выключить опцию |
| Выпадающий список | Select | Выберите один вариант |
| Цвет | color picker | Клик → выбор цвета |
| Число | Поле с стрелками | Позиция, размер шрифта |

### 3.4. Загрузка изображений (общий алгоритм)

1. Нажмите **«Загрузить»** или **«Выбрать файл»** рядом с нужным блоком
2. Выберите файл на компьютере (JPG, PNG; на сервере конвертируется в WebP)
3. В окне кадрирования (**Cropper**) выделите нужную область
4. Нажмите **«Применить»** / **«Сохранить»**
5. Дождитесь появления превью и URL файла
6. Не забудьте нажать глобальное **«Сохранить изменения»**

**Требования к изображениям (ТЗ Приложение №3):**

- Баннеры: до 1920×1080, рекомендуется ≤ 300 КБ до загрузки
- Новости: до 800×600
- Формат: JPG, PNG (конвертируется автоматически)
- Цветовое пространство: sRGB

### 3.5. Переключение темы CMS

Кнопка ☀️/🌙 в правом верхнем углу переключает светлую и тёмную тему интерфейса админки. На публичный сайт это не влияет.

## 4. ГЛАВНАЯ СТРАНИЦА

**CMS:** Главная страница  
**Публичный URL:** `index.html`

### 4.1. Hero-слайдер (главный баннер)

Главный экран сайта может содержать до **8 слайдов**, переключаемых автоматически или вручную.

**Поля каждого слайда:**

| Поле | Описание | Совет |
|------|----------|-------|
| Заголовок (title) | Крупный текст на баннере | Enter — новая строка |
| Подзаголовок (subtitle) | Дополнительный текст | Можно оставить пустым |
| Фон (background) | Изображение баннера | Загрузить через Cropper |
| Цвет заголовка | HEX (#ffffff) | Контраст с фоном |
| Позиция заголовка | Top, Left (px) | Настройка визуально |
| Размер шрифта | Число или пусто (авто) | — |

**Как добавить слайд:**

1. Прокрутите до блока «Hero-слайды»
2. Нажмите **«+ Добавить слайд»**
3. Заполните поля
4. **Сохранить изменения**
5. Обновите `index.html` в браузере (Ctrl+F5)

### 4.2. Карточки услуг (serviceCards)

Четыре (или более) плитки с направлениями деятельности:

| Поле | Пример |
|------|--------|
| title | «Обучение» |
| desc | Краткое описание |
| link | `obuchenie.html` |
| variant | green / peach / purple / blue |
| icon | Иконка услуги |
| external | Ссылка внешняя (да/нет) |

### 4.3. Промо-баннер

Блок акции/курса под hero:

- **title** — заголовок акции
- **date** — даты проведения
- **link** — ссылка при клике
- **image** — изображение баннера

### 4.4. Партнёры

Логотипы организаций-партнёров. Для каждого:

- **alt** — альтернативный текст (важно для SEO и доступности!)
- **image** — логотип (рекомендуется PNG с прозрачностью)

### 4.5. Отзывы

Блок отзывов клиентов:

- **text** — текст отзыва (до ~243 символов на карточке)
- **nameLines** — имя (массив строк)
- **roleLines** — должность

### 4.6. Блок консультации

- **photos** — фотографии специалистов

### 4.7. Социальные сети

Ссылки на Max, Telegram, VK:

- **id** — идентификатор (max, tg, vk)
- **label** — подпись
- **href** — URL страницы

### 4.8. Виджет чата

Настройка приветственных сообщений (если не используется только Bitrix24):

- **operatorName** — имя оператора
- **operatorAvatar** — аватар
- **welcomeMessages** — массив приветствий
- **autoReplies** — автоответы

## 6. ЛЕНДИНГ «ОБУЧЕНИЕ»

**CMS:** Страница Обучение + Календарь курсов  
**Публичный URL:** `obuchenie.html`

### 6.1. Разделы страницы Обучение

| Блок на сайте | Редактирование в CMS |
|---------------|---------------------|
| Hero-баннер | Страница Обучение → Hero |
| Навигационные карточки | Nav Cards |
| Календарь курсов | Автоматически из реестра курсов |
| Поиск курсов | Фильтры courseSearch |
| Преподаватели | Вкладка «Преподаватели» |
| Форма записи | Настраивается через курсы + Bitrix |

### 6.2. Управление календарём курсов (ВАЖНО)

Полный реестр курсов ведётся на отдельной странице:

**https://zakupki.tatar/new_design/admin-obuchenie-courses.html**

#### 6.2.1. Добавление нового курса

1. Откройте **admin-obuchenie-courses.html**
2. Войдите (те же логин/пароль, что и в основной CMS)
3. Нажмите **«+ Добавить курс»**
4. Заполните форму:

| Поле | Обязательно | Описание |
|------|-------------|----------|
| Название | ✅ | Полное название программы |
| Дата начала | ✅ | Календарь Flatpickr |
| Формат | ✅ | Очный / Заочный |
| Длительность (дней) | ✅ | Минимум 1 |
| Стоимость | — | Например: «23 700 ₽» |
| Аудитория | ✅ | Физ. лица / Юр. лица (хотя бы одно) |
| Опции | — | 44-ФЗ, Заказчик, Поставщик и др. |
| Описание | — | WYSIWYG-редактор |
| Bitrix24 ФЛ | — | ID формы для физ. лиц |
| Bitrix24 ЮЛ | — | ID формы для юр. лиц |

5. Нажмите **«Сохранить»**
6. При успехе:
   - курс появится в таблице;
   - день подсветится в календаре на `obuchenie.html`;
   - **автоматически создастся лид в Bitrix24** (мероприятие);
   - в статусе может отобразиться «лид Bitrix24 #XXXXX»

#### 6.2.2. Редактирование курса

1. В таблице курсов нажмите **«Редактировать»**
2. Внесите изменения
3. **Сохранить**

#### 6.2.3. Удаление курса

1. **«Удалить»** → подтвердите в диалоге
2. Курс исчезнет из календаря

#### 6.2.4. Настройка форм Bitrix24

Для кнопок «Записаться» (ФЛ/ЮЛ) на карточке курса:

1. В форме курса найдите **«Bitrix24 — физ. лица»** / **«юр. лица»**
2. Нажмите **«Настроить»**
3. Вставьте скрипт формы из Bitrix24 **или** введите формат: `1048 / 6zzb7x` (ID / sec)
4. **Сохранить** в модалке → **Сохранить** курс

### 6.3. Запись на курс (со стороны посетителя)

На `obuchenie.html` посетитель:

1. Открывает календарь → выбирает дату с курсом
2. Нажимает **«Записаться»**
3. Заполняет: имя, телефон, e-mail, компания (для ЮЛ), источник
4. Отправляет → заявка уходит в Bitrix24 как лид

## 9. РАЗДЕЛ «НОВОСТИ»

**CMS:** Страница Новости  
**Публичный URL:** `news.html`

### 9.1. Просмотр списка новостей

В CMS отображается таблица/список всех новостных записей с датой, заголовком и статусом.

### 9.2. Создание новости

1. CMS → **Страница Новости**
2. Нажмите **«Добавить новость»** (или аналогичную кнопку)
3. Заполните поля:

| Поле | Описание | Рекомендация |
|------|----------|--------------|
| **Заголовок** | Название новости | До 255 символов, информативный |
| **Слаг (slug)** | URL-идентификатор | `novost-o-kurse-2026`, латиница |
| **Анонс (excerpt)** | Краткое описание | 1–3 предложения для превью |
| **Текст (body)** | Полный текст | WYSIWYG, абзацы, списки |
| **Миниатюра** | Изображение превью | 800×600, alt-текст |
| **Дата** | Дата публикации | Формат даты |
| **Статус** | draft / published | Только published видна на сайте |
| **Теги** | Ключевые слова | Через запятую |

4. **Сохранить изменения**
5. Откройте `news.html` — новость должна появиться в ленте

### 9.3. Редактирование и снятие с публикации

- **Редактировать** — изменить поля, сохранить
- **Черновик** — установите status = draft, новость скроется с сайта
- **Удалить** — безвозвратное удаление (подтвердите действие)

### 9.4. Поиск и фильтрация на сайте

Публичная страница новостей поддерживает поиск и фильтрацию по тегам (при наполнении указывайте релевантные теги: «44-ФЗ», «Обучение», «ЭТП»).

### 10.1. Структура: дерево групп

Документы организованы в **иерархическое дерево**:

```
📁 Нормативные акты
   📁 44-ФЗ
      📄 Положение о закупках.pdf
      📄 Регламент.docx
   📁 223-ФЗ

   📄 Памятка заказчику.pdf
```

### 10.2. Создание группы

1. CMS → **Страница Документы**
2. Нажмите **«+ Группа»**
3. Введите название группы
4. При необходимости — выберите родительскую группу (вложенность)
5. Сохраните

### 10.3. Добавление документа

1. Выберите группу в дереве
2. Нажмите **«+ Документ»** / **«Загрузить файл»**
3. Выберите файл (PDF, DOCX, XLSX — до 20 МБ)

5. Сохраните

### 10.4. Редактирование и перемещение

- Переименование — поле названия
- Перемещение — смена родительской группы (drag-and-drop или селектор)
- Удаление — кнопка удаления с подтверждением

### 10.5. Требования к файлам (ТЗ Приложение №3)

| Параметр | Значение |
|----------|----------|

| Имя файла | Латиница, цифры, `_` (при загрузке переименовывается автоматически) |

## 12. РАБОТА С МЕДИА-ФАЙЛАМИ

### 12.1. Изображения

**Поддерживаемые форматы загрузки:** JPG, PNG, GIF, WebP  
**Результат на сервере:** WebP (оптимизированный)

**Рекомендуемые размеры:**

| Назначение | Разрешение | Макс. вес |
|------------|------------|-----------|
| Hero-баннер | 1520×420 – 1920×1080 | до 1.2 МБ |
| Карточка новости | 800×600 | 300 КБ |
| Иконка / логотип | SVG или 200×200 | 50 КБ |
| Партнёр | 400×400 | 100 КБ |

**Alt-текст (ТЗ п. 5.3.2):** для каждого значимого изображения указывайте описание для поисковиков и незрячих пользователей.

### 12.2. Документы

Загрузка через `api/upload-file.php` из интерфейса «Документы» или соответствующих полей.

### 12.3. Видео

**Прямая загрузка на сервер не рекомендуется** (ТЗ Приложение №3).

Используйте:

- YouTube, RuTube, VK Video
- Вставка через iframe или поле «URL видео» в CMS

### 12.4. Что делать, если изображение «не загружается»

1. Проверьте размер файла (уменьшите в Photoshop / Squoosh)
2. Проверьте формат (не HEIC, не TIFF)
3. Попробуйте другой браузер
4. Обратитесь к администратору (возможна проблема с правами `uploads/`)

## 13. SEO И МЕТА-ДАННЫЕ

### 13.1. Требования ТЗ (п. 5.3.1)

Для каждой страницы должны настраиваться:

| Параметр | Назначение |
|----------|------------|
| **Title** | Заголовок вкладки браузера, главный для SEO |
| **Description** | Описание для сниппета в поиске |
| **Keywords** | Ключевые слова (опционально) |
| **Canonical** | Канонический URL |
| **Robots** | noindex, nofollow при необходимости |

### 13.2. Рекомендации по заполнению

**Title:**

- Длина 50–60 символов
- Формат: «Услуга — АО ЦРЗ РТ» или «Новость | Центр развития закупок РТ»
- Уникальный для каждой страницы

**Description:**

- 150–160 символов
- Содержит призыв к действию
- Не дублирует title дословно

**Заголовки на странице:**

- Один **H1** на страницу
- Логическая иерархия H2 → H3

### 13.3. Open Graph

При публикации ссылки в соцсетях (VK, Telegram) используются изображение и заголовок страницы. Убедитесь, что у главных лендингов заданы качественные баннеры.

## 15. ЧАСТО ЗАДАВАЕМЫЕ ВОПРОСЫ

**В: Я изменил текст, но на сайте старое. Почему?**  
О: Не нажата кнопка «Сохранить изменения». Либо очистите кэш браузера (Ctrl+F5).

**В: Можно ли отменить изменения?**  
О: Если не сохраняли — перезагрузите страницу CMS. Если сохранили — восстановление только из бэкапа администратором.

**В: Сколько слайдов можно на главной?**  
О: До 8 hero-слайдов.

**В: Курс добавлен, но в Bitrix24 нет лида.**  
О: Сообщите администратору — проверят webhook. Попробуйте открыть курс и сохранить снова.

**В: Как вставить таблицу в новость?**  
О: Через WYSIWYG или вставьте как изображение таблицы.

**В: Работает ли CMS с телефона?**  
О: Интерфейс адаптивен, но для объёмного редактирования рекомендуется компьютер.
